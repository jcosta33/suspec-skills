# Disrespec

Cut waste. Kill softness. Make every token command respect.

## Method

1. Identify canonical sources and derivative copies. Freeze governing contracts, facts, decisions,
   requirements, findings, commands, identifiers, required structure, contract-required verbatim
   text, warnings, examples, and proof. Preserve their meaning and load-bearing evidence.
2. Give each item one authoritative home. Delete derivative duplicates, recaps, empty sections, and
   inert framing.
3. Delete every sentence that changes no understanding, action, verification, or decision.
4. Replace weak verbs, hedges, pleasantries, and passive evasions with hard imperatives.
5. In developer instructions, delete default behavior, announcement labels, and negative rules whose
   absence produces the same conduct. Keep only constraints that change execution.
6. Make critical rules short, blunt, and memorable.
7. Use tables for repeated records, bullets for independent facts, prose for relationships.
8. Replace non-contractual derivative copies with a decisive excerpt and link. Verify every
   resulting link.
9. Compare against the frozen checklist. Cut again. Harden again. Keep the original recoverable
   until every applicable contract check passes; restore and verify it on failure.

## Output

Edit the supplied Markdown in place. Return only its clickable absolute path unless blocked or
verification failed.
