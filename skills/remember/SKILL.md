# Remember

Save signal. Drop residue.

## Method

1. Collect candidates from the completed run.
2. Classify each as ephemeral, personal durable, team durable, or severe.
3. Reject guesses. A remembered claim needs durable evidence, scope, and expiry bounds where
   behavior can change.
4. Reject credentials, tokens, private keys, personal/customer data, confidential prompts, and raw
   output containing them. Redact safely or use an approved restricted channel.
5. When native harness memory exists, save one personal claim per memory under a searchable title.
   Include evidence, where it applies, and what it does not establish.
6. When native memory does not exist, create no substitute file or store. Leave personal-only
   observations ephemeral.
7. Route team facts through project-owned channels: decisions to the decision process, reproduced
   defects to issue tracking, invariants to authorized tests, operational facts to runbooks.
8. Escalate severe correctness, security, privacy, data-loss, or compliance issues before close.

Do not duplicate a team record into personal memory unless the memory links to it and adds a useful
retrieval cue. Correct or remove native memory when later evidence disproves it.

After every lesson is accounted for, ask about a transient artifact set only when Remember is its
final consumer, the set has reached true lifecycle close, and no earlier disposition prompt
occurred. Otherwise leave the single prompt to the artifact owner or later final consumer. Never
issue a second prompt. When eligible, ask once: Delete, Leave, or Promote. Recommend from state,
explain each option in one sentence, include the complete path set, and include `Other`. Never
choose for the human or treat silence as Leave.
