# Body anatomy

A skill body is executable conditioning. Soft prose leaves room to ignore the method.

Use this order:

1. `# Name`.
2. One non-negotiable stance when it changes conduct.
3. `## Method` with ordered imperatives.
4. `## Artifact` for artifact contracts.
5. `## Verify` for distinct proof work.
6. `## Decision` for human acceptance.
7. `## Output` for constrained handoff.
8. `## Boundaries` for refusals or failure behavior.
9. `## Close` for artifact disposition.

`Method` is mandatory. Omit every other section without payload. Never rename phases or repeat the
description.

- Keep only instructions that change execution.
- Use hard verbs.
- Delete encouragement, scene-setting, default behavior, repeated rationale, and conclusions.
- State each fact once.
- Move optional domain detail into one-level-deep bundled references.
- Keep every reference inside the skill directory.
- Split the body before it reaches the official 500-line guidance.

Numbering, rationales, examples, and anti-patterns are tools, not ritual. Keep them only when they
change execution. Commentary is not a method.
