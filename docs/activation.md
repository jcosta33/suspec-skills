# Activation

The frontmatter `description` is the discovery contract. It must tell the harness when loading the
body is worth its context cost.

## Rule

A description states:

1. the action and target;
2. observable triggers, including explicit invocation words;
3. the dangerous shortcut it refuses; and
4. adjacent task types it skips.

Open with the work. Use `ALWAYS apply` only for a real automatic trigger. Never depend on a sibling
skill name to route work. Name the task type instead. Keep the description below the Agent Skills
specification limit and verify that neighboring descriptions do not claim the same request.

## Test

For each description, construct:

- a direct invocation that must trigger;
- an implicit request that must trigger;
- a near neighbor that must not trigger; and
- a request where loading it would waste context.

Rewrite until each outcome is obvious from metadata alone.
